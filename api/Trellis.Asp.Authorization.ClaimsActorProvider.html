Remarks

Register as scoped in DI via AddClaimsActorProvider(IServiceCollection, Action<ClaimsActorOptions>?). Claim mapping is controlled by ClaimsActorOptions.

This provider assumes authentication has already occurred. It throws InvalidOperationException if no authenticated user exists.

Extending for nested or computed claims. The default mapping is flat (see the ClaimsActorOptions remarks). For provider-specific shapes — JSON-object claims, dotted paths, claims that need to be merged with an external store, or permissions derived from multiple raw claims — subclass this provider and override GetCurrentActorAsync(CancellationToken). The HttpContextAccessor and Options properties are protected precisely to support this. EntraActorProvider is a worked example.
