Remarks

Supported syntax:

  • ### Title separators (title becomes test-case display name).
  • # @name foo metadata line — binds this request's response under key foo for later {{foo.response.*}} substitution.
  • # @expect status: 201 / 2xx / 200-299 — optional status assertion.
  • # @expect header: ETag — required response header (must be present and non-empty).
  • # @parity: status-only — cross-host parity directive.
  • @variable = value — file-level variables usable via {{variable}}.
  • {{named.response.body.jsonPath}} — dotted-path substitution against a prior captured response body (JSON).
  • {{named.response.headers.Name}} — header substitution against a prior captured response.

Expectation semantics consumed by HttpFileAssertions: if no @expect is present on a request, the runner still asserts the status falls in the non-error range 100-399. This catches regressions (e.g. a happy-path endpoint suddenly 500'ing) without forcing authors to annotate every 2xx request.
