Table of Contents

Namespace Trellis.Testing.AspNetCore.Http

Classes

ExpectedOutcome

Expected outcome parsed from # @expect pragmas in a .http file.

HttpFileAssertionException

Raised by HttpFileAssertions when a request's expectations are not met. Carries a descriptive message identifying the offending request by title so failures are easy to triage in CI.

HttpFileAssertions

Verifies that an HttpFileResult satisfies its attached ExpectedOutcome.

HttpFileParser

Parses a .http file (VS Code REST Client / Visual Studio format) into a sequence of HttpFileRequest values ready to be replayed by HttpFileRunner.

HttpFileRequest

A single HTTP request parsed from a .http file (VS Code REST Client / Visual Studio HTTP file format).

HttpFileResult

Result of replaying a single HttpFileRequest.

HttpFileRunner

Executes a parsed .http file against an HttpClient, threading a ScenarioContext through the requests so named responses can be referenced via {{name.response.body.*}} substitutions.

HttpFileTheoryData

Produces [MemberData]-compatible test cases from a .http file. Returns IEnumerable<object[]> rather than an xUnit-specific TheoryData<T> so this helper carries no xUnit dependency from the library itself; consumer test projects can freely wrap with TheoryData<HttpFileRequest> if they prefer the typed variant.

ScenarioContext

Mutable state shared across a sequence of HTTP file requests. Captures each # @name foo-annotated response so later requests can reference {{foo.response.body.x}} / {{foo.response.headers.Name}}.
